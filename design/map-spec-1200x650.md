# Map Spec 1200x650

*Created: 2026-05-08*  
*Status: Production Draft*

---

## 1. Purpose

这份文档把当前确认的 `C` 路线地图概念，反推成适配项目现有 `1200 x 650` 画布的落地规范。

目标不是还原一张插画，而是定义：
- 地图各区域在当前游戏里的尺寸关系
- 哪些位置服务玩法
- 哪些位置服务视觉叙事
- 后续地图资源如何拆分

---

## 2. Current Runtime Constraints

当前项目主画布：
- 画布尺寸：`1200 x 650`
- 左侧领地区：`0 - 840`
- 右侧工厂区：`840 - 1200`
- 顶部 HUD：约 `55px`
- 底部 HUD：约 `55px`

实际主要战场可视高度：
- `y = 55` 到 `y = 595`

因此地图设计必须优先考虑这块中间可战斗区域。

---

## 3. Horizontal Region Spec

### A. 圣域高地核心区
- `x = 0 - 250`
- 定位：玩家家园、核心守护区、最高地势
- 功能：
  - 放置纯净核心
  - 建立第一层生命氛围
  - 给予玩家视觉安全感

视觉要求：
- 地势最高
- 草地最鲜亮
- 花草最丰富，但不要盖住交互单位
- 有柔和高地边缘和平台感

玩法要求：
- 核心周围至少保留一圈相对干净的阅读空间
- 不要让大树或装饰物压到核心判定和敌人接近判读

### B. 左侧防守活动区
- `x = 250 - 500`
- 定位：主要树木种植和守卫者近身作战区
- 功能：
  - 树木中前排主要分布区
  - 守卫者拦截和移动的主要空间

视觉要求：
- 仍属于高地或缓坡上部
- 植被丰富，但必须留出清晰空地
- 地表要比圣域区更“可战斗”

玩法要求：
- 这是最重要的可读性区域之一
- 地面不能过碎
- 树木建议偏分组式分布，而不是铺满

### C. 中央对抗焦点区
- `x = 500 - 760`
- 定位：生态衰败过渡带，地图视觉中心
- 功能：
  - 展示自然与污染的对抗
  - 成为玩家最常注意的战斗焦点带

视觉要求：
- 明确的由绿到黄再到裸土的过渡
- 一条较窄的溪流或河道穿过
- 必须有 1 个明确 crossing 记忆点：小桥 / 石桥 / 踏石
- 这是整张图最应该“被记住”的区域

玩法要求：
- 中央焦点不能被画得太满
- 需要给守卫者和敌人视觉交锋留出读图空间
- 即使背景有桥，也不要让桥的透视破坏单位辨识

### D. 污染推进区
- `x = 760 - 960`
- 定位：污染明显扩散、敌人进入领地前的压境带
- 功能：
  - 让敌人看起来像从污染中压过来
  - 强化由生态到工业的衰败感

视觉要求：
- 草地基本退化
- 裸土、灰黄地面、紫灰污染斑显著增加
- 可见少量污染裂缝、污泥流痕、死草

玩法要求：
- 这是敌人进入玩家关注区前的准备带
- 背景需要清晰衬托敌人轮廓，不宜过花

### E. 工厂低地区
- `x = 960 - 1200`
- 定位：敌方来源地，工业污染主场
- 功能：
  - 承担工厂叙事
  - 承担敌人出生视觉来源

视觉要求：
- 地势最低、最平
- 工厂、管道、烟囱、喷口、废桶轮廓明确
- 颜色以灰棕、灰紫、橙色警示为主

玩法要求：
- 虽然这里敌人较多，但仍要保证怪物轮廓清楚
- 工厂建筑不要复杂到盖住刷新和移动读图

## 4. Vertical Composition Spec

### Upper Band
- `y = 55 - 180`
- 作用：远景化、地形轮廓化、给地图呼吸感
- 内容建议：
  - 高地边缘
  - 稀疏树顶
  - 少量工厂远景元素

要求：
- 上边区域不应堆太多高对比元素
- 避免和顶部 HUD 打架

### Mid Combat Band
- `y = 180 - 470`
- 作用：主战斗读图区
- 内容建议：
  - 纯净核心
  - 大部分树木
  - 守卫者活动空间
  - 主要敌人推进路线
  - 中央桥/河道焦点

要求：
- 这里是地图设计优先级最高的区域
- 所有地形、装饰、桥和污染痕迹，都必须为战斗清晰度让路

### Lower Band
- `y = 470 - 595`
- 作用：补充地貌层次，建立地势和路径完整性
- 内容建议：
  - 河流尾段
  - 低地泥痕
  - 坡脚
  - 少量装饰地块

要求：
- 避免与底部 HUD 区域抢视觉焦点

## 5. Key Object Placement

### Pure Core
- 推荐落点：`(130, 325)` 到 `(170, 325)` 区间
- 原则：
  - 保留在左侧高地
  - 不要太贴左边界
  - 不要太贴中央战斗区

### Tree Planting Field
- 建议主要分布区：
  - `x = 220 - 620`
  - `y = 120 - 520`

设计建议：
- 按自然斑块组织视觉
- 但底层逻辑仍可维持当前网格
- 可以“视觉自然、逻辑规则”

### Main Bridge / Crossing
- 推荐落点：`x = 620 - 700`
- 推荐纵向位置：`y = 260 - 380`

作用：
- 成为地图中部视觉记忆点
- 强化中央对抗线

### Enemy Emergence Belt
- 建议分布区：
  - `x = 980 - 1160`
  - `y = 120 - 520`

设计建议：
- 不只一个出生喷口
- 可以做成 2-3 组污染裂缝 / 管道口 / 废液喷口
- 敌人像从工业污染系统中被释放出来

## 6. Terrain And River Rules

### Height Logic
- 左侧必须明显高于右侧
- 不能靠单纯颜色表示高差
- 必须增加：
  - 台地边缘
  - 缓坡明暗
  - 地块层叠
  - 溪流流向

### River Rule
- 河流宽度控制在“装饰性且可读”
- 推荐视觉宽度：
  - 窄处约 `28 - 40px`
  - 宽处约 `50 - 70px`

河道目标：
- 是空间组织线
- 不是强分割线
- 必须服务中央战斗叙事

## 7. Asset Slice Recommendation

不建议未来只做一张死背景图。

推荐拆分为 4 层：

### Layer 1: Base Terrain
- 草地
- 裸土
- 荒地
- 高低地表
- 大体河道

用途：
- 作为整张地图基础

### Layer 2: Landmark Features
- 纯净核心地纹
- 主桥/踏石
- 工厂主建筑轮廓
- 污染喷口

用途：
- 提升地图记忆点

### Layer 3: Decorative Overlay
- 花丛
- 灌木
- 小石块
- 污染裂痕
- 边缘污泥

用途：
- 调节密度
- 方便后期增减，不影响底图

### Layer 4: Optional Foreground / FX Support
- 轻烟雾
- 污染漂浮层
- 河面光斑
- 圣域微光

用途：
- 只在需要时叠加，提升氛围

---

## 8. Production Sequence

建议按这个顺序推进：

1. 先定 `1200x650` 地图结构规范
2. 再产出 1 张接近该比例的正式地图背景概念稿
3. 再决定是整图绘制，还是按图层拆资源
4. 最后替换当前项目中的地图背景并调整树木区、敌人区、工厂区视觉坐标

## 9. What This Means For Code

当前代码里这些区域后续值得一起调整：
- `CONFIG.territory.width`
- `CONFIG.factory.x / width`
- 树木网格 `offsetX / offsetY / cellW / cellH`
- 核心位置
- 工厂敌人生成可视区域

注意：
- 这一步先不急着改代码
- 先让视觉规范稳定，再做坐标调整，返工更少
