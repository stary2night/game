# 金币经济系统 (Coin Economy System)

> **Status**: Draft
> **Author**: Game Designer
> **Last Updated**: 2026-05-07
> **Implements Pillar**: 自然与人的协力

## Summary

击杀怪物获得金币，金币用于购买武器升级和树木升级两条路线。经济系统是战斗系统和生态系统的桥梁，迫使玩家在火力提升与生态投资之间做出策略决策。

> **Quick reference** — Layer: `Core` · Priority: `MVP` · Key deps: `wave-system.md, combat-weapon-system.md, ecology-system.md`

---

## Overview

金币是游戏唯一资源。每击杀一个怪物掉落金币，金币可在商店购买更强武器或升级树木。由于金币总量有限，玩家必须权衡：优先火力（武器）还是优先防线（树木），这一决策贯穿整个游戏，体现了"个人力量"与"自然力量"的博弈。

---

## Player Fantasy

玩家应感受到：每一枚金币都是我努力战斗的成果，花在哪里是我的战略抉择，不同的消费路线带来不同的游戏体验。

## Detailed Design

### Core Rules

1. 初始金币：`STARTING_COINS = 50`（用于第一波前可以做初始决策）
2. 金币无上限
3. 金币不随波次重置，全程积累
4. 金币以浮动文字形式从怪物位置飞向HUD金币计数器
5. 商店可在任何时候打开（不暂停游戏）
6. 购买行为立即生效

### 购买选项

**武器购买（一次性）**

| 商品 | 价格 | 备注 |
|------|------|------|
| 火焰大剑 | 150金 | 替换当前武器 |
| 寒冰大剑 | 200金 | 替换当前武器 |
| 冰火法杖 | 300金 | 替换当前武器 |

**树木升级（累计）**

| 升级等级 | 价格 | 效果 |
|----------|------|------|
| 普通橡树 → 强化橡树 | 100金 | TREE_HP ×1.5，CO₂吸收×1.5 |
| 强化橡树 → 神圣生命树 | 200金 | TREE_HP ×2.0（vs原始），CO₂吸收×2.0，治愈速率×1.5 |

注：树木升级对**所有现有和未来种植**的树木生效（全局升级）。

### 预期金币收入

完整5波理论总收入（无失误）：
- Wave 1: 8×10 = 80金
- Wave 2: 6×10 + 4×18 = 132金
- Wave 3: 4×10 + 4×18 + 4×15 = 172金（×1.2倍率≈206金）
- Wave 4: 6×10 + 4×18 + 4×15 + 1×120 = 312金（×1.5倍率≈468金）
- Wave 5: 6×18 + 6×15 + 2×120 + 1×300 = 798金（×2.0倍率≈1596金）

加初始50金，理论总金币约 **2532金**

可购买全部商品：150+200+300+100+200 = **950金**，理论上完全可以全部解锁。

### States and Transitions

无状态机，金币系统为简单计数器。

### Interactions with Other Systems

| System | Interaction |
|--------|-------------|
| 波次入侵系统 | 怪物死亡事件触发金币奖励 |
| 战斗武器系统 | 购买武器时消耗金币并通知武器系统切换 |
| 生态循环系统 | 购买树木升级时修改树木参数 |

## Formulas

### 金币奖励（见波次系统）

```
coin_reward = BASE_COIN * WAVE_MULTIPLIER[wave] * ENEMY_TYPE_MULTIPLIER
```

### 树木升级后属性

```
upgraded_tree_hp = BASE_TREE_HP * UPGRADE_HP_MULTIPLIER[level]
upgraded_absorb = BASE_CO2_ABSORB * UPGRADE_ABSORB_MULTIPLIER[level]
```

| Level | HP Multiplier | Absorb Multiplier |
|-------|--------------|-------------------|
| 1（普通橡树） | 1.0× | 1.0× |
| 2（强化橡树） | 1.5× | 1.5× |
| 3（神圣生命树） | 2.0× | 2.0× |

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 购买价格恰好等于当前金币 | 允许购买，余额为0 | 精准规划应被奖励 |
| 已购买最高级树木，再次点击升级 | 按钮显示"已满级"，无法购买 | 防止重复消费 |
| 已购买冰火法杖后再想买火焰大剑 | 允许，金币扣除，武器切换 | 武器随时可切换 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 波次入侵系统 | 本系统依赖 | 接收怪物死亡金币事件 |
| 战斗武器系统 | 武器系统依赖本系统 | 购买时调用金币检查和扣除 |
| 生态循环系统 | 生态系统依赖本系统 | 升级树木时调用参数修改接口 |

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| STARTING_COINS | 50 | 0-150 | 玩家第一波前有更多选择 | 玩家必须先打怪才能买东西 |
| 火焰大剑价格 | 150 | 80-250 | 需要更多怪物才能买 | 早期即可获得 |
| 树木升级1价格 | 100 | 60-200 | 生态投资更昂贵 | 早期可以强化生态 |
| WAVE_MULTIPLIER[4] | 1.5× | 1.2-2.0× | 后期金币更富裕 | 后期也相对拮据 |

## Visual/Audio Requirements

| Event | Visual Feedback | Audio Feedback | Priority |
|-------|----------------|---------------|----------|
| 怪物死亡掉金币 | 金色硬币图标从怪物位置飞向HUD | 金属碰撞音效 | HIGH |

| 达到可购买新物品金额 | 商店按钮轻微金色发光提示 | 无（避免干扰） | LOW |

## Acceptance Criteria

- [ ] 初始拥有50金币
- [ ] 每种怪物死亡掉落正确数量金币（含波次倍率）
- [ ] 购买武器正确消耗金币并切换武器
- [ ] 购买树木升级正确消耗金币并修改所有树木参数
- [ ] 金币不足时无法购买，有明确UI反馈
- [ ] 已满级树木无法重复购买
